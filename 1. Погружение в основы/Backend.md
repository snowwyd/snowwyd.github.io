*Изучение этого раздела познакомит тебя с написанием первых сервисов на go. Начнешь с простого REST API через фреймворк Gin, а закончишь проектом с несколькими микросервисами и использованием gRPC.*

*Впрочем, если половину слов ты тут не понял - ничего страшного, все узнаешь в процессе. Для мотивации: после того, как изучишь этот раздел - ты будешь уже на уровне junior программиста на Go и все, что тебе останется - подготовиться к собеседованиям.*

Твоя задача - идти по порядку, который будет описан ниже, а также здесь будет чек-лист того, что тебе предстоит сделать уже самому.

Также при написании сервисов каждый раз возвращайся к статье [Работа с Git и GitHub](../Работа%20с%20Git%20и%20GitHub/)

1. [REST API](../REST%20API/)
2. [gRPC](../gRPC/)
3. [Связывание микросервисов](../Связывание%20микросервисов/)
4. [Подключение MongoDB к сервису](../Подключение%20MongoDB%20к%20сервису/)

Чек-лист:
- [ ] Посмотри материалы и повтори сам, опираясь на них то, что было сделано в них
- [ ] Придумай свой сервис на REST API и запрограммируй его, подключив Postgres
- [ ] Придумай свой микросервис на gRPC и также сделай его с Postgres
- [ ] Свяжу два предыдущих микросервиса с собой, чтобы они работали вместе 
- [ ] Напиши gRPC микросервис, который будет работать с MongoDB
- [ ] Залей все сервисы в отдельные репозитории на Github, причем поработай с коммитами, заливая все постепенно
