# Настройка GitHub Pages для отображения всех Markdown файлов

## Что было сделано:

1. **`_config.yml`** - конфигурация Jekyll для обработки всех .md файлов
2. **`index.md`** - главная страница сайта  
3. **`_layouts/default.html`** - шаблон для красивого отображения
4. **`Gemfile`** - зависимости для Jekyll

## Быстрая настройка (3 шага):

### Шаг 1: Включи GitHub Pages

1. Перейди в **Settings** → **Pages** в твоем репозитории
2. В разделе **"Source"** выбери:
   - **Branch**: `main` (или `master` - в зависимости от твоей ветки)
   - **Folder**: `/ (root)`
3. Нажми **Save**

### Шаг 2: Подожди 1-2 минуты

GitHub автоматически соберет сайт с помощью Jekyll. Ты увидишь зеленую галочку в разделе Pages, когда сайт будет готов.

### Шаг 3: Открой сайт

Твой сайт будет доступен по адресу:
```
https://[твой-username].github.io/[название-репозитория]/
```

Например: `https://username.github.io/sobesedovaniya-s-nulya/`

## Как это работает:

✅ **Jekyll** автоматически конвертирует все `.md` файлы в HTML  
✅ Каждый `.md` файл становится отдельной страницей  
✅ Ссылки между файлами работают автоматически  
✅ Поддерживается кириллица в названиях файлов и директорий  

## Доступ к файлам:

После настройки все твои `.md` файлы будут доступны по URL:
- `README.md` → `https://username.github.io/repo/README.html`
- `1. Погружение в основы/Go язык.md` → `https://username.github.io/repo/1.%20Погружение%20в%20основы/Go%20язык.html`

**Важно**: В URL используй URL-кодирование (`%20` для пробелов, `%28`/`%29` для скобок)

## Формат ссылок в Markdown:

Используй относительные пути с URL-кодированием и расширением .html:
```markdown
[Go язык](1.%20Погружение%20в%20основы/Go%20язык.html)
[Этап 1](1.%20Погружение%20в%20основы/Этап%201.%20Погружение%20в%20основы.html)
```

## Если что-то не работает:

1. **Проверь логи сборки**: Settings → Pages → посмотри последнюю сборку
2. **Убедись, что файлы закоммичены**: все файлы должны быть в репозитории
3. **Подожди еще немного**: иногда GitHub Pages обновляется с задержкой

## Локальный запуск (опционально):

Если хочешь проверить сайт локально перед публикацией:

```bash
# Перейди в директорию репозитория
cd "СОБЕСЕДОВАНИЯ (С НУЛЯ)"

# Установи Ruby (если еще не установлен)
# Windows: скачай с rubyinstaller.org
# Mac: уже установлен
# Linux: sudo apt-get install ruby ruby-dev

# Установи Bundler
gem install bundler

# Установи зависимости
bundle install

# Запусти локальный сервер
bundle exec jekyll serve
```

Сайт будет доступен на `http://localhost:4000`

## Альтернатива (если Jekyll не подходит):

Если возникнут проблемы, можно использовать:
- **MkDocs** - через GitHub Actions
- **VuePress** - через GitHub Actions  
- **Docusaurus** - через GitHub Actions

Но для начала попробуй Jekyll - он самый простой и уже встроен в GitHub Pages!
