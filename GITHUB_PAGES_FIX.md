# Инструкция по исправлению проблемы с GitHub Pages

## Проблема
Ошибка 404 на главной странице.

## Решение

### Вариант 1: Статический сайт (рекомендуется)

1. **Создан файл `.nojekyll`** - это отключает обработку Jekyll
2. **index.html** - статический HTML файл, который будет работать напрямую
3. Все `.md` файлы нужно конвертировать в `.html` вручную или использовать другой инструмент

### Вариант 2: С Jekyll (текущая настройка)

1. **index.html** исключен из обработки Jekyll в `_config.yml`
2. **index.html** - статический файл, который GitHub Pages будет отдавать напрямую
3. Все `.md` файлы обрабатываются Jekyll

## Проверка настроек GitHub Pages

1. Перейди в **Settings** → **Pages** в твоем репозитории
2. Убедись, что:
   - **Source**: выбран branch (main/master)
   - **Folder**: `/ (root)`
3. Сохрани изменения

## После коммита

1. Подожди 1-2 минуты
2. Проверь сайт по адресу: `https://username.github.io/repository-name/`
3. Если не работает, проверь логи сборки в Settings → Pages

## Если все еще не работает

Попробуй создать файл `.nojekyll` в корне репозитория (пустой файл) - это отключит Jekyll полностью и будет использоваться только статический HTML.
